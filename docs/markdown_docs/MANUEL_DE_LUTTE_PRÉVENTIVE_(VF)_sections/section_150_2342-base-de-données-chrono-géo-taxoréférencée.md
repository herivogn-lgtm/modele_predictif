## **2.3.4.2. Base de données chrono-, géo-, taxoréférencée** 

Le logiciel Microsoft[®] Access a été retenu pour gérer les bases de données et seules les données issues de ces bases seront utilisables par le SIG-LMC (Arc GIS). Les données gérées sous Microsoft[®] Excel ne sont pas directement compatibles avec Arc GIS car ce logiciel n’est qu’un tableur et non un gestionnaire de base de données. 

Le principe de base de l’archivage des données est de constituer, pour chaque locuste, des bases de données relationnelles qui soient strictement chrono- et géoréférencée de manière à pouvoir replacer chaque événement acridien dans son contexte spatio-temporel : 

– lors des analyses en temps réel mais aussi lors des synthèses diachroniques ou synchroniques et pour la constitution de référentiels permettant de comparer une situation particulière à la situation moyenne observée sur une durée aussi longue que possible (30 ans pour constituer une normale) ; 

– pour l’élaboration des cartes de fréquence (Ould Babah, 2003 ; Babah Ebbe, 2008). 

Outre les données analytiques collectées sur le terrain, il est recommandé de prévoir, pour la base acridienne, des tables de données synthétiques facilitant l’accès à des chrono- et géoréférencements complexes ou à des informations globales relatives à la phase ou à la phénologie de chaque acridien.
